# Security Policy

## Reporting a Security Issue

Please do not report security vulnerabilities through public GitHub issues.

If you believe you have found a security issue involving Virtual Research Institute,
the website, the Ask VRI backend, or any related infrastructure, please report it
privately through the official contact information listed on the Virtual Research
Institute website.

When reporting a security issue, please include:

- A clear description of the issue
- Steps to reproduce it, if possible
- Any relevant URLs, screenshots, or error messages
- Your contact information, if you would like a response

Virtual Research Institute will review reports in good faith and take appropriate
action where needed.

## Scope

This policy applies to the public website, source code, backend Worker code,
configuration files, and related materials in this repository.

## Prohibited Activity

Please do not:

- Access, modify, or delete data that does not belong to you
- Attempt to bypass authentication, authorization, rate limits, or usage controls
- Disrupt service availability
- Publicly disclose vulnerabilities before Virtual Research Institute has had a
  reasonable opportunity to review and address them
